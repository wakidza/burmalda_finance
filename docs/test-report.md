## 2. Стратегия тестирования

- **Модульные тесты** проверяют бизнес‑логику сервисов в изоляции (категории,
  операции, бюджеты, аналитика). Используется реальная in‑memory SQLite, что
  заодно проверяет и репозитории.
- **Интеграционные тесты** проверяют HTTP‑API целиком через `TestClient`
  FastAPI: коды ответов, валидацию, формат ошибок, сериализацию.
- Каждый тест работает с **изолированной БД в памяти** (фикстуры в
  `tests/conftest.py`), поэтому тесты независимы и повторяемы.

## 3. Покрытые сценарии

### 3.1. Категории (модульные + API)
- создание категории;
- запрет дубликата (имя+тип) → `409 ConflictError`;
- одинаковое имя при разных типах разрешено;
- получение несуществующей → `404 NotFoundError`;
- переименование, в т. ч. конфликт при переименовании;
- удаление;
- валидация пустого имени → `422`.

### 3.2. Операции (модульные + API)
- создание операции, дата по умолчанию (сегодня);
- запрет несоответствия типа операции и категории → `422 ValidationError`;
- несуществующая категория → `404`;
- отрицательная/нулевая сумма → `422` (валидация Pydantic);
- изменение суммы; смена типа с перепроверкой категории;
- удаление;
- фильтрация по типу и периоду.

### 3.3. Бюджеты (модульные + API)
- бюджет только для категории расходов (income → `422`);
- несуществующая категория → `404`;
- запрет дубля бюджета на месяц → `409`;
- расчёт исполнения: «в пределах лимита» и «перерасход» (spent/remaining/percent/over_budget);
- изменение лимита снимает признак перерасхода.

### 3.4. Аналитика (модульные + API)
- сводка за месяц и за год (доходы/расходы/баланс);
- разбивка по категориям, сортировка по убыванию;
- помесячный тренд: ровно 12 точек, корректные значения по месяцам.

### 3.5. Служебное
- `/api/health` отвечает `ok`;
- корень `/` отдаёт фронтенд.

## 4. Ручное тестирование

Проводилось на запущенном сервере (`uvicorn app.main:app`) с засеянной БД.

## 5. Найденные и исправленные ошибки

| № | Описание | Причина | Исправление |
|---|----------|---------|-------------|
| BUG‑1 | Предупреждение `StarletteDeprecationWarning` при импорте кода ошибок | Константа `HTTP_422_UNPROCESSABLE_ENTITY` переименована в новой версии Starlette | Заменена на числовой литерал `422` с комментарием (`app/core/error_handlers.py`) |
| BUG‑2 | Неиспользуемое/некорректное выражение при извлечении месяца в аналитике | Черновой код с `func.cast(..., type_=None)` | Упрощено до `func.strftime("%m", ...)`, единое выражение в `group_by` |
| BUG‑3 | Опечатка в импорте обработчиков ошибок (`status_placeholder`) на раннем этапе | Невнимательность при наборе | Файл переписан с корректным импортом `status` |
| BUG‑4 | `@app.on_event("startup")` помечен как устаревший | Старый API инициализации FastAPI | Переведено на современный `lifespan` |

Все перечисленные дефекты устранены; повторный прогон тестов — зелёный.

## 6. Известные ограничения (не дефекты)

- Суммы хранятся как `float` (округление до 2 знаков). Для строгой финансовой
  арифметики рекомендуется `Decimal`/целые копейки — осознанный компромисс
  (см. ТЗ, раздел «Ограничения»).
- Помесячная агрегация использует `strftime` (SQLite‑специфично).
- Приложение однопользовательское (без аутентификации).
